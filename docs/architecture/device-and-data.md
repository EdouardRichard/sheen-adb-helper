# 设备与本地数据现状

> 类型：当前主干事实
> 依据：HEAD 的 `:core:data`、设备/概览/设置 Feature 及测试。

## 模块归属

- `:core:data` 拥有设备档案模型、DataStore 仓库、档案变更、文本导出与临时数据清理。
- `:feature:devices` 负责地址输入、连接/配对、最近设备、重命名、删除和进程内诊断展示。
- `:feature:overview` 通过 `AdbSessionManager` 加载设备概览，并只在前台刷新动态指标。
- `:feature:settings` 展示版本、许可证/隐私信息、系统设置帮助和清除全部本地数据入口。

## 数据归属

- 设备档案保存稳定 ID、显示名、主机、调试端口、首次/最近连接时间、本机标记和身份引用；按最近连接排序。
- 档案元数据保存在 DataStore；损坏的版本化记录被拒绝，不解释为有效档案。
- ADB 主机身份不属于 DataStore，由 `:core:adb` 管理；最后一个相关档案删除或清除全部数据时清除身份。
- 概览、进程快照、Shell、Logcat、应用列表和操作结果不进入档案或 DataStore。

## 公共行为与限制

- 概览尽力提供系统、构建、CPU、内存、存储、电池、温度、运行时间和网络字段；不可可靠获得的字段保持缺失，不伪造默认值。
- 动态指标由页面前台状态控制刷新；离开、断开或切换 Session 后停止。
- 文本导出写入用户选择的 SAF 目标，不申请所有文件访问，也不缓存导出副本。
- 清除全部本地数据覆盖档案、身份、偏好和临时文件；产品不提供账号或云端恢复。

## 安全边界

- 主机地址属于本地元数据，日志和测试输出必须脱敏；设备详情和身份材料不得上传。
- 当前 Manifest 只声明 `android.permission.INTERNET`；权限事实以 [`docs/权限矩阵.md`](../权限矩阵.md) 为准。
