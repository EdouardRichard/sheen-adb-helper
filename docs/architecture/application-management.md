# 应用管理现状

> 类型：当前架构事实
> 依据：HEAD 的 `:core:adb`、`:feature:apps`、`:app` 及对应测试。

## 当前能力

- `:feature:apps` 显示被控端当前 Android 用户的第三方应用包列表，以包名为主标题。
- 支持大小写不敏感搜索、全部/已启用/已禁用的内存筛选和手动刷新；不轮询、不持久化。
- 可靠可得时显示版本/安装器字段；不可可靠获得时明确降级，不推测应用名或图标。
- 用户可对单个允许目标发起强制停止、禁用或重新启用；每次修改均需一次性确认。

## 核心与安全边界

- 当前用户探测、第三方包列表、包名校验、命令构造、Session 快照、修改和后置验证都位于 `:core:adb`。
- 修改目标必须来自同 Session 最近的安全快照，属于当前用户且非系统包；包名采用保守语法并有容量上限。
- 本机 `127.0.0.1` Session 禁止操作 Sheen 自身；未知状态、系统包、列表外或过期 Session 目标均拒绝。
- 强停只返回“请求已接受”语义；启用状态修改只有后置查询符合预期才返回已验证成功。
- 修改后超时、取消、EOF 或掉线可能产生结果未知；UI 要求重新连接/刷新，不宣称成功、失败或回滚。

## 状态、数据与限制

- 列表、当前用户、搜索/筛选、确认和操作结果只在内存中；断开或 Session 变化清空旧状态。
- 诊断事件不含包名、用户 ID、命令或输出；应用管理不新增 Manifest 权限、外部依赖或 DataStore 格式。
- 不支持安装、卸载、系统应用、跨用户、批量、定时、后台、Root、Shizuku、设备管理员或无障碍自动化。
