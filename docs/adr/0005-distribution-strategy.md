# ADR 0005：默认分发策略

- 状态：`accepted`
- 日期：2026-07-18
- 负责人：Sheen ADB 助手项目

## 背景

v0.01 将小米应用商店列为分发目标，并在生命周期规范、权限矩阵和发布门禁中保留了特定商店审核要求。项目负责人现明确要求移除应用商城上架目标：后续除非项目负责人主动提出，否则不再以小米应用商店或其他应用商店上线为目标。

分发策略属于 [`../生命周期与AI协作规范.md`](../生命周期与AI协作规范.md) 规定的 ADR 事项。已发布版本的历史需求和结项事实不得回写，因此 v0.01 文档中的当时目标保持原样。

## 候选方案

1. 继续以小米应用商店为默认目标：与项目负责人最新指令冲突，不采用。
2. 改为某个其他应用商店：没有对应需求，不采用。
3. 默认面向开源仓库、自行构建和用户自主安装；未来仅在明确需求下新增特定分发渠道：采用。

## 决策

- 项目默认通过公开源码、可复现构建说明和用户自主安装方式分发。
- 不为未被明确提出的应用商店预留 SDK、权限、条件功能、渠道包或审核材料。
- 不再把小米商店审核演示、截图、说明或特定审核规则作为通用版本完成门禁。
- 若未来重新提出任何应用商店上架目标，必须更新当前版本需求、权限矩阵和本 ADR，评估渠道规则、功能等价、签名、隐私与审核材料。
- 开源仓库继续使用 Apache-2.0；依赖许可证白名单、安全、隐私、权限最小化和危险操作确认规则不因分发渠道变化而降低。
- v0.01 文档保留其历史分发目标，不修改历史结论。

## 理由

该方案符合最新产品方向，减少为未承诺渠道进行的设计和审核工作，同时保持可审计的开源发布、安全门禁和未来重新评估渠道的能力。

## 影响

- 生命周期规范中的“权限与商店合规”改为通用“权限与分发合规”。
- 权限矩阵的变更问题改为面向已明确的实际分发渠道。
- v0.02 SDD 不以小米或其他应用商店审核为范围或验收门禁。
- 不允许因为没有商店审核而扩大权限、降低危险操作确认或忽略许可证冲突。

## 回滚方式

若未来需要上架特定应用商店，由项目负责人明确目标渠道，新增本 ADR 的修订记录或替代 ADR，并同步需求文档、权限矩阵、隐私政策、构建/签名策略和验收材料。不得仅在实现中加入渠道特例。
