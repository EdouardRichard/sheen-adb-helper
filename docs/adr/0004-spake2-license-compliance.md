# ADR 0004：spake2-java 许可证合规处置

- 状态：采用
- 日期：2026-07-15
- 负责人：Sheen ADB 助手项目

## 背景

Kadb 2.1.1 的 Android 运行时通过 JitPack 引入 `com.github.Flyfish233:spake2-java:1.0.5`，用于 Android 11+ 无线 ADB 的 SPAKE2 配对。

Phase 0 与 v0.01 依赖文档原先将该依赖记录为 Apache-2.0。结项复核发现，该精确版本的发布 POM 声明 GPL-3.0-or-later，其 `1.0.5` 标签中的 [`LICENSE`](https://github.com/Flyfish233/spake2-java/blob/1.0.5/LICENSE) 也是 GPLv3。该许可证不属于项目允许的 Apache-2.0、MIT、BSD-2-Clause、BSD-3-Clause 白名单。

本 ADR 只登记冲突与决策边界，不对许可证兼容性作法律结论，也不授权修改业务代码。

## 候选方案

1. 取得精确版本的白名单许可证依据：由权利人提供可公开审计、明确覆盖 `1.0.5` 发布物的 Apache-2.0、MIT 或 BSD 授权；复核通过后同步依赖清单和应用内许可证展示。
2. 替换或排除该传递依赖：在 `:core:adb` 内改用白名单许可证的 SPAKE2/ADB 配对实现，新增协议决策和回归计划，并重新验证配对、取消、超时与资源关闭。
3. 放宽许可证白名单：这与当前 `AGENTS.md` 硬性规则冲突；除非项目负责人先明确变更仓库约束、开源与分发策略，否则不作为可执行方案。

## 临时决策

- 在方案 1 或方案 2 完成前，不把 v0.01 标记为发布合规通过。
- 保留现有实现用于已完成验收事实的可追溯记录，但不以更正文档代替依赖处置。
- 本轮不修改依赖、协议实现、应用内许可证 UI 或发布产物。

## 影响

- v0.01 的功能真机验收结论不被否定，但发布合规结项保持暂缓。
- `docs/第三方依赖审查.md` 与 `docs/第三方依赖与许可证.md` 必须如实显示当前冲突。
- 若选择替换实现，变更只能发生在 `:core:adb`，并须重新执行相关自动化与真机验收。

## 完成门禁

- 精确发布物的许可证证据可审计，且符合仓库白名单；或替换实现已完成依赖、协议和许可证审查。
- Version Catalog、解析依赖、依赖文档、NOTICE 和应用内许可证展示一致。
- Android 11+ 配对、HyperOS 3 / Android 16 配对、取消、超时、错误码和资源关闭回归通过。

## 回滚方式

本 ADR 当前不引入代码或依赖变更。若后续候选方案不可行，维持发布合规暂缓状态，不通过删除记录或改写许可证名称回滚。
