## 1. 背景、目标与成功标准

### 1.1 背景

v0.01 已建立纯本地的单活跃 ADB Session、无线配对、设备概览、Shell、只读进程和前台 Logcat 能力，并已完成 HyperOS 3 / Android 16 功能真机验收。v0.02 在不改变连接、安全和隐私边界的前提下，增加有限的被控端应用管理能力。

### 1.2 版本目标

v0.02 必须让用户能够：

1. 查看被控端**当前 Android 用户**下的第三方应用包列表；
2. 按包名搜索并按启用状态筛选；
3. 对单个允许操作的第三方应用发起强制停止；
4. 对单个允许操作的第三方应用执行禁用或重新启用；
5. 在 ROM 拒绝、会话断开、结果无法确认时获得准确、可行动的结构化反馈。

### 1.3 成功标准

- 应用管理命令全部封装在 `:core:adb` 的命名能力中，Feature/UI 不出现业务 Shell 命令。
- 任何操作都绑定发起时的 session ID；旧会话结果不得污染新设备 UI。
- 默认不持久化包列表、包状态、操作历史或命令输出。
- 不新增依赖，不新增 Manifest 权限，不读取主控端应用列表。
- 不允许对系统应用、预装系统包、其他 Android 用户或批量目标执行修改操作。
- 自动化和真机验收覆盖正常、空、降级、拒绝、超时、取消、掉线和会话切换。

## 2. 范围

### 2.1 纳入 v0.02

| ID | 能力 | 类型 | 交付级别 |
|---|---|---|---|
| SCOPE-01 | 当前用户第三方应用列表 | 读取 | 必须 |
| SCOPE-02 | 包名搜索、启用/禁用筛选、手动刷新 | 读取 | 必须 |
| SCOPE-03 | 可靠字段详情与明确降级 | 读取 | 必须 |
| SCOPE-04 | 单包强制停止 | 短时破坏性 | 必须 |
| SCOPE-05 | 单包禁用与重新启用 | 可恢复但高风险 | 必须 |
| SCOPE-06 | 操作确认、结构化错误、结果验证 | 安全 | 必须 |

### 2.2 明确延期

- APK 安装、更新、降级、split APK、APKS/XAPK/AAB。
- 应用卸载、为当前用户移除、保留数据卸载。
- 应用名称和图标的 APK 提取或资源解析。
- 跨重启保存“Sheen 禁用过哪些包”的恢复账本。
- 工作资料、访客、次用户、双开空间的显式管理。

上述能力进入未来版本前必须重新进行需求、协议、权限、数据与风险评审。安装还必须新增二进制传输与 Package Manager Session ADR。

### 2.3 明确排除

- 系统应用、预装系统包、关键系统组件的强停、禁用、启用或卸载。
- 清除应用数据、修改权限、修改默认应用、修改电池策略。
- 批量、定时、后台、规则驱动或无人值守操作。
- 应用搜索、下载、分发、应用市场或远程更新源。
- 读取主控端应用列表，或声明 `QUERY_ALL_PACKAGES`/包可见性来实现被控端能力。
- Root、Shizuku、无障碍服务、设备管理员或绕过系统限制。

## 3. 用户与使用场景

### 3.1 目标用户

沿用 v0.01：了解 ADB 风险的 Android 高级用户、开发者和设备维护人员。

### 3.2 主要用例

#### UC-01 查看应用

1. 用户连接设备并进入“应用管理”。
2. 页面读取当前 Android 用户 ID，再读取该用户的第三方应用列表。
3. 页面显示包名、启用状态和可靠可得的可选字段。
4. 用户可搜索、筛选或手动刷新。

#### UC-02 强制停止

1. 用户选择一个允许操作的第三方包。
2. 页面显示包名、目标设备、当前用户和不可恢复影响。
3. 用户明确确认后发起操作。
4. UI 只表达“请求已执行/被拒绝/结果未知”，不得宣称应用将永久保持停止。

#### UC-03 禁用与重新启用

1. 用户选择当前已启用或已禁用的第三方包。
2. 页面说明禁用可能影响通知、后台任务和默认处理能力。
3. 用户明确确认后发起单包操作。
4. `:core:adb` 在操作后重新读取包状态；只有状态符合预期时显示成功。

### 3.3 非正常场景

- 当前用户无法识别：列表整体失败，不回退到所有用户。
- ROM 不支持某个查询字段：保留列表并标注字段不可用。
- 包在确认期间被卸载或状态改变：返回“目标状态已变化”，刷新列表。
- 操作过程中断开或超时：显示“结果未知，请重新连接并刷新确认”。
- 切换设备：取消旧操作并清空旧包列表、筛选结果和待确认对话框。

## 4. 功能需求

### 4.1 应用列表

- **FR-LIST-001**：列表必须只查询当前 Android 用户，禁止默认查询 `all` 或省略会导致全用户操作的用户参数。
- **FR-LIST-002**：默认且唯一管理范围为当前用户第三方应用；系统应用可以不出现在列表中。
- **FR-LIST-003**：每项必须包含稳定 `packageName` 和 `enabledState`。
- **FR-LIST-004**：可选字段包括 `versionCode`、`versionName`、安装器标识；仅在设备可靠提供时展示。
- **FR-LIST-005**：v0.02 不承诺应用名称和图标。UI 以包名作为主标题，不伪造名称。
- **FR-LIST-006**：支持包名大小写不敏感的本地搜索。
- **FR-LIST-007**：支持“全部、已启用、已禁用”筛选；筛选不得再次发起不必要的 ADB 查询。
- **FR-LIST-008**：首次进入页面时加载一次；用户可手动刷新。离开页面不轮询。
- **FR-LIST-009**：列表为空时区分“没有第三方应用”和“ROM 输出无法解析”。
- **FR-LIST-010**：包列表仅驻留内存；断开、session ID 变化或进程结束后清除。

### 4.2 强制停止

- **FR-STOP-001**：只能从已加载的、当前 session 的第三方应用项发起。
- **FR-STOP-002**：每次操作都必须二次确认，不提供“以后不再提示”。
- **FR-STOP-003**：确认文案必须包含包名，并说明可能中断前台任务、下载、通知和未保存工作。
- **FR-STOP-004**：本机 `127.0.0.1` 会话中，必须禁止对 Sheen 自身包 `com.sheen.adbhelper` 发起强制停止。
- **FR-STOP-005**：命令被目标端接受后显示“已发送强制停止请求”；不得声称目标不会自动重启。
- **FR-STOP-006**：系统/策略拒绝、包不存在和会话失效必须映射为不同结构化错误。

### 4.3 禁用与重新启用

- **FR-STATE-001**：只允许对当前用户第三方应用执行包级禁用/启用，不提供组件级操作。
- **FR-STATE-002**：每次禁用和启用都必须二次确认。
- **FR-STATE-003**：禁用确认必须说明应用入口、通知、后台任务和关联功能可能不可用。
- **FR-STATE-004**：UI 使用“禁用”和“重新启用”；“重新启用”只表示修改 enabled state，不承诺恢复运行状态或数据。
- **FR-STATE-005**：操作前必须确认目标仍属于发起时 session、当前用户和第三方范围。
- **FR-STATE-006**：操作后必须重新查询目标状态；状态符合预期才返回已验证成功。
- **FR-STATE-007**：本机连接中禁止禁用 Sheen 自身。
- **FR-STATE-008**：v0.02 不保存操作归属；允许用户重新启用任何列表中当前已禁用、且符合安全边界的第三方应用。

### 4.4 刷新、取消和并发

- **FR-OPS-001**：同一时刻只允许一个活跃 ADB Session，且每个 session 同一时刻只执行一个应用管理高层操作。
- **FR-OPS-002**：加载或修改期间禁用其他应用管理按钮，并提供取消入口。
- **FR-OPS-003**：默认超时：列表 15 秒；强制停止 10 秒；状态修改与验证合计 15 秒。
- **FR-OPS-004**：取消、超时或底层异常必须完成流、命令和临时对象清理。
- **FR-OPS-005**：取消或超时可能发生在目标已执行之后；此时结果必须为 `OutcomeUnknown`，引导重新连接/刷新，不得标记为“未执行”。
- **FR-OPS-006**：离开页面取消正在进行的列表读取；已经发出的修改操作按核心层清理规则结束，并禁止结果写回已离开的旧 session 状态。

## 5. 非功能需求

### 5.1 安全与隐私

- **NFR-SEC-001**：包列表、包名和应用状态按敏感设备数据处理，默认不落盘、不上传、不进入 Android Log 或第三方日志。
- **NFR-SEC-002**：诊断事件禁止包含真实包名、命令文本、命令输出、用户 ID 或安装路径；只允许阶段、结果、结构化代码和脱敏目标。
- **NFR-SEC-003**：业务命令参数不得接受自由文本；包名必须来自当前列表并再次通过包名语法校验。
- **NFR-SEC-004**：不得为“方便”加入系统包开关、隐藏高级模式或绕过确认的 Debug 后门。
- **NFR-SEC-005**：清除所有本地数据不新增应用管理数据清理项，因为 v0.02 不持久化该数据。

### 5.2 可靠性

- **NFR-REL-001**：所有结果携带发起 session ID；ViewModel 必须丢弃不匹配结果。
- **NFR-REL-002**：解析失败不得导致页面崩溃或产生半真半假的列表。
- **NFR-REL-003**：修改操作采用“前置校验 → 命名操作 → 后置验证”的单一高层能力。
- **NFR-REL-004**：系统应用识别不可靠时，默认拒绝修改，而不是扩大范围。
- **NFR-REL-005**：一次失败不得破坏连接管理器后续重试能力；若底层必须关闭 session，UI 必须明确显示已断开。

### 5.3 性能与容量

- **NFR-PERF-001**：列表在 IO dispatcher 执行，不阻塞主线程。
- **NFR-PERF-002**：内存列表上限为 20,000 项；超过时返回结构化容量错误，不创建无界集合。
- **NFR-PERF-003**：搜索和筛选在已加载快照上完成，目标是普通千项列表交互无明显卡顿。

### 5.4 可访问性与适配

- **NFR-UI-001**：简体中文优先，跟随系统主题和字体缩放。
- **NFR-UI-002**：颜色不是启用状态、风险或成功的唯一表达。
- **NFR-UI-003**：危险操作按钮具有可读语义，确认对话框焦点顺序正确。
- **NFR-UI-004**：手机竖屏、横屏和平板沿用现有单栏/扩展布局策略。

## 6. 架构设计

### 6.1 模块视图

```text
:app
  ├─ 组装 AppsViewModel
  ├─ 注册“应用管理”目的地
  └─ 观察全局连接状态

:feature:apps
  ├─ AppsScreen / AppsRoute
  ├─ AppsViewModel
  ├─ immutable AppsUiState
  └─ 纯 UI 搜索、筛选、确认状态

:core:adb
  ├─ RemoteApplication 等项目自有模型
  ├─ AdbSessionManager 命名能力
  ├─ 应用命令能力探测与解析
  ├─ session 归属、超时、取消、错误映射
  └─ Kadb 适配器（不向外泄漏类型）

:core:data
  └─ v0.02 无新增职责
```

依赖方向保持 `app/feature → core`。`:feature:apps` 不依赖其他 Feature；`:core:adb` 不依赖 Compose、导航或 Feature。

### 6.2 新增模块

新增 `:feature:apps`。不将应用管理并入 `:feature:processes`，因为包状态、风险确认和错误语义与只读进程快照不同。

模块只使用现有 Compose、Lifecycle、Coroutines、`:core:adb` 和 `:core:ui` 依赖，不新增第三方库。

### 6.3 App 导航

抽屉顺序调整为：

1. 设备列表 / 首页
2. 设备概览
3. 应用管理
4. Shell 终端
5. 进程监控
6. Logcat
7. 设置与隐私

“应用管理”依赖已连接状态；未连接时可见、置灰并提示“请先连接设备”。断开时若当前位于该页面，导航回首页。

## 7. 核心接口与模型设计

以下为语义契约，开发时可以在不改变语义的前提下调整 Kotlin 命名。

```kotlin
data class RemoteApplication(
    val packageName: String,
    val userId: Int,
    val enabledState: RemoteApplicationEnabledState,
    val versionCode: Long? = null,
    val versionName: String? = null,
    val installerPackage: String? = null,
    val isSystem: Boolean,
)

enum class RemoteApplicationEnabledState {
    ENABLED,
    DISABLED,
    UNKNOWN,
}

data class ApplicationSnapshot(
    val sessionId: String,
    val userId: Int,
    val applications: List<RemoteApplication>,
    val unavailableFields: Set<ApplicationField>,
    val degradedReason: String? = null,
)

sealed interface ApplicationMutationResult {
    data class Verified(val application: RemoteApplication) : ApplicationMutationResult
    data object RequestAccepted : ApplicationMutationResult
    data class OutcomeUnknown(val reason: AdbError) : ApplicationMutationResult
}
```

`AdbSessionManager` 新增命名能力：

```kotlin
suspend fun listApplications(timeout: Duration): AdbOperationResult<ApplicationSnapshot>

suspend fun forceStopApplication(
    packageName: String,
    expectedSessionId: String,
    timeout: Duration,
): AdbOperationResult<ApplicationMutationResult>

suspend fun setApplicationEnabled(
    packageName: String,
    enabled: Boolean,
    expectedSessionId: String,
    timeout: Duration,
): AdbOperationResult<ApplicationMutationResult>
```

### 7.1 命令边界

- 具体 `cmd package`、`pm`、`am` 命令只定义在 `:core:adb` 内部能力层。
- 首次列表读取进行能力探测，优先使用目标 Android 版本支持的稳定命令；不依赖单一厂商私有输出。
- 当前用户 ID 必须由核心层获取并校验为非负整数。
- 包名必须匹配 Android 包名的保守语法，并且存在于同 session 最近一次安全快照中。
- 任何命令非零退出码、stderr 拒绝文本或后置状态不一致都不得返回已验证成功。

### 7.2 操作阶段与错误

`AdbOperationStage` 增加：

- `APPLICATIONS_LIST`
- `APPLICATION_FORCE_STOP`
- `APPLICATION_SET_ENABLED`
- `APPLICATION_VERIFY`

项目自有错误至少区分：

| 技术代码 | 含义 | 用户下一步 |
|---|---|---|
| `ADB_APP_CURRENT_USER_UNAVAILABLE` | 无法确认当前用户 | 不执行任何修改；检查 ROM 支持 |
| `ADB_APP_LIST_UNSUPPORTED` | 列表命令不受支持或无法解析 | 查看降级说明；保留其他功能 |
| `ADB_APP_PACKAGE_NOT_FOUND` | 包已不存在 | 刷新列表 |
| `ADB_APP_TARGET_NOT_ALLOWED` | 系统包、自身包或范围外目标 | 不提供绕过入口 |
| `ADB_APP_POLICY_REJECTED` | ROM/企业策略拒绝 | 在被控端检查管理策略 |
| `ADB_APP_STATE_VERIFY_FAILED` | 操作后状态不符 | 刷新并以设备实际状态为准 |
| `ADB_APP_OUTCOME_UNKNOWN` | 超时/掉线导致结果不确定 | 重新连接并刷新确认 |

技术详情和诊断事件不得包含包名或原始拒绝文本。

## 8. UI 与状态设计

### 8.1 `AppsUiState`

状态至少包含：

- `isConnected`、`sessionId`
- `isLoading`、`activeOperation`
- `applications`、`query`、`filter`
- `degradedReason`、`unavailableFields`
- `error`
- `pendingConfirmation`
- `operationNotice`

### 8.2 页面状态

| 状态 | 表达 |
|---|---|
| 未连接 | “请先连接设备”，无操作入口 |
| 初始加载 | 进度和取消按钮 |
| 内容 | 搜索、筛选、列表、刷新 |
| 空 | 明确“当前用户没有可显示的第三方应用” |
| 降级 | 列表可用，缺失字段单独说明 |
| 可恢复错误 | 原因、下一步、重试 |
| 不可恢复错误 | 说明 ROM/策略不支持，不循环重试 |
| 修改中 | 目标包、操作名、取消入口；其他修改按钮禁用 |
| 结果未知 | 醒目提示重新连接/刷新，以设备实际状态为准 |

### 8.3 列表项

- 主标题：包名。
- 次要信息：启用状态；版本字段可得时展示。
- 不展示伪造图标或由包名猜测的应用名。
- 操作入口必须是可发现按钮或菜单，不依赖滑动手势。
- `UNKNOWN` 状态下不显示禁用/启用按钮，只允许刷新。

### 8.4 确认策略

每次修改操作都显示确认对话框；默认焦点不落在确认按钮上。对话框必须显示：

1. 操作名称；
2. 完整包名；
3. 当前设备显示名和当前用户 ID；
4. 主要后果；
5. “取消”和明确动作文本。

## 9. 数据与隐私设计

| 数据 | 分级 | 存储 | 清理 |
|---|---|---|---|
| 包名、版本、启用状态 | 敏感设备数据 | 仅内存 | 断开、session 切换、进程退出 |
| 当前用户 ID | 敏感上下文 | 仅内存 | 同上 |
| 搜索词、筛选 | 临时 UI 数据 | 仅当前 ViewModel | 页面/进程销毁 |
| 操作结果 | 临时 UI 数据 | 仅内存 | session 切换或用户关闭提示 |
| 操作历史 | 不收集 | 无 | 不适用 |

v0.02 不修改 DataStore 格式，不修改设备档案，不增加导出能力。

## 10. 权限、依赖与分发

### 10.1 Manifest

v0.02 预计仍只使用现有 `android.permission.INTERNET`。不得新增：

- `QUERY_ALL_PACKAGES` 或主控端包可见性声明；
- 存储、所有文件访问、通知、前台服务；
- 设备管理员、无障碍、Root 相关能力。

被控端应用管理来自用户已经明确建立的 ADB 会话，不等同于读取主控端已安装应用。

### 10.2 依赖

v0.02 不新增依赖。若现有依赖无法实现本 SDD，必须暂停并提出替代方案，不得擅自添加库。

ADR 0004 的许可证冲突必须在对外发布前解决；不得通过改写许可证名称绕过。

### 10.3 分发

本项目默认面向开源仓库、自行构建和用户自主安装，不以小米应用商店或其他应用商店上架为目标。除非未来需求由项目负责人明确提出，不开展特定商店的功能裁剪、审核材料或上架适配。

这不降低法律、安全、隐私、权限最小化、危险操作确认和第三方许可证要求。

## 11. 生命周期、取消与资源清理

- 页面离开时取消只读列表加载并清除待确认对话框。
- session 变化立即取消 Feature 作业并清空快照。
- 核心操作必须在 `finally` 中关闭临时流/任务；不得关闭不属于该操作的全局资源。
- 若底层取消只能通过关闭活跃 ADB client 实现，连接状态必须由 `AdbSessionManager` 发布为断开，UI 不得继续显示已连接。
- 修改命令发出后发生取消、超时或 EOF 时，返回 `OutcomeUnknown`，不能声称回滚。
- 禁止后台继续应用管理操作；不使用服务、通知或 WorkManager。

## 12. 测试设计

### 12.1 单元测试

#### `:core:adb`

- 当前用户解析：正常、空、非数字、负数、多行、ROM 噪声。
- 列表解析：已启用、已禁用、可选字段缺失、重复包、空、畸形、超过上限。
- 系统/第三方识别与保守拒绝。
- 包名验证：正常、边界、空白、Shell 元字符、换行、超长。
- 命名命令契约：明确 `--user`，无 UI 拼接输入。
- session ID 匹配、切换后丢弃旧结果。
- 强停接受、拒绝、包消失、超时、取消、结果未知。
- 禁用/启用：前置状态、后置验证、验证不一致、策略拒绝。
- 诊断事件不包含包名、命令、输出或用户 ID。

#### `:feature:apps`

- 加载、成功、空、降级、错误、断开、取消、结果未知状态。
- 搜索和筛选纯逻辑。
- 每次危险操作必须确认。
- 系统包、自身包、未知状态无修改入口。
- session 变化清空列表、确认和旧通知。

#### `:app`

- “应用管理”菜单顺序。
- 离线置灰与读屏语义。
- 断开时返回首页。

## 14. 风险与缓解

| 风险 | 等级 | 缓解 |
|---|---|---|
| 禁用重要第三方应用影响设备使用 | 高 | 仅第三方、单包、每次确认、系统识别不明即拒绝 |
