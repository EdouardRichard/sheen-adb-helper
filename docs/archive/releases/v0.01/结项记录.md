## 2. 分类结论

### 已通过

- v0.01 在 HyperOS 3 / Android 16 基线上的总体功能真机验收已由项目负责人确认通过。
- 全部 Gradle 模块保持 `minSdk 30`、`compileSdk 36`，应用保持 `targetSdk 36`。
- Release 合并 Manifest 仅声明 `android.permission.INTERNET`，与 [`权限矩阵.md`](../../../权限矩阵.md) 一致；SAF 导出不新增 Manifest 权限。
- Version Catalog 中的直接依赖版本与 v0.01 依赖清单一致。
- 结项审阅时可读取的本地自动化报告为 24 项测试无失败、Lint 无错误；这些构建目录已被忽略，不作为仓库内持久化验收证据。

### 已知限制

- 旧式 ADB TCP/IP `:5555` 仅尽力兼容，不保证所有设备或 ROM 可用。
- ROM 的 `ps`、`dumpsys`、网络与设备字段输出存在差异；无法可靠解析时明确降级，不伪造成功或默认值。
- 配对库公开 API 会临时构造 JVM `String`；不持久化、不记录且不保留引用，但字符串无法原地覆盖。
- 无线调试端口可能变化；重启或系统状态变化后可能需要用户重新核对端口。

### 延期到后续版本

- v0.02 候选：应用列表及应用管理能力。
- v0.03 候选：设备文件浏览、双向传输与 APK 提取。
- v0.04 候选：二维码配对、局域网发现及更多分析能力。
- 上述内容均不属于 v0.01，不以结项补充名义回写历史需求。

### 仍需补证据

- 被验收安装包对应的完整提交、构建类型及可公开校验值。
- v0.01 清单逐项的“通过 / 已知限制 / 未执行”结果；总体通过不替代逐项记录。
- Android 11 与非小米设备覆盖情况，以及有设备时旧式 `:5555` 的结果。
- ADR 0001 所列错误配对码、取消与资源释放、抓包/中间人负向验证的逐项脱敏结论。
- Phase 0 是否曾独立执行；当前只能确认其核心场景已纳入 v0.01 验收范围。

## 3. 发布合规阻断

结项复核发现，Kadb 2.1.1 的运行时传递依赖 `spake2-java 1.0.5` 在其发布 POM 与精确标签的 `LICENSE` 中声明为 GPL-3.0-or-later，而原依赖文档将其记录为 Apache-2.0。该许可证不在当前仓库白名单内。

在取得该精确版本可审计的白名单许可证依据，或以白名单实现替换并完成必要回归前：

- v0.01 功能真机验收结论保持“通过”；
- v0.01 发布合规结论保持“暂缓”；
- 不得把许可证名称直接改写为白名单许可证来消除冲突；
- 应用内现有第三方许可证展示也不能视为最终合规清单。

处置边界与候选方案见 [`ADR 0004`](../../../adr/0004-spake2-license-compliance.md)。

## 4. 最终结论

v0.01 已完成并通过总体功能真机验收，但仓库证据链尚不完整，且第三方许可证阻断仍不可忽略。功能版本已结项；许可证/对外发布合规尚未通过。
