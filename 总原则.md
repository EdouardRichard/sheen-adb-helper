## 必须实现的功能

### 1. 应用基础与导航

实现简约主界面：

- 顶部左侧为三横线抽屉菜单。
- 首页核心是一个 `IP:端口` 输入框和相邻“连接”按钮。
- 顶部持续显示当前状态：未连接、连接中、等待设备授权、已连接、配对中、断开中、错误。
- 抽屉菜单顺序固定：
  1. 设备列表 / 首页
  2. 设备概览
  3. Shell 终端
  4. 进程监控
  5. Logcat
  6. 设置与隐私
- 未连接时，设备概览、Shell、进程和 Logcat 入口可见但置灰，提示“请先连接设备”。
- 平板和横屏采用合理双栏/扩展布局，不能仅简单拉伸手机布局。
- 所有可点击控件具备无障碍语义和足够触控区域。

### 2. 地址、连接、配对和设备档案

地址输入必须支持：

- IPv4：`192.168.1.2:5555`
- 主机名：`device.local:端口`
- IPv6：`[fe80::1]:端口`

禁止自动扫描局域网、mDNS 发现或附近设备扫描。

连接流程：

- 首页始终优先尝试直接连接。
- 已配对 Android 11+ 设备或旧式 `:5555` 设备可以直接连接。
- 旧式 ADB 等待 RSA 授权时，提示用户到被控端确认，不伪造“已连接”。
- 仅在认证/未配对相关失败时，显示“使用配对码”入口；网络不可达、超时等错误不能误导为配对失败。
- 配对页面输入 `IP:配对端口` 与 6 位配对码；成功后引导输入调试端口或在已知可用时自动连接。
- 抽屉提供“本机连接”，预填 `127.0.0.1`，用户仍需填写当前调试端口。
- 连接中可取消；连接、配对、断开均须有超时、取消、finally 清理和可复制的脱敏技术详情。
- 错误至少区分：地址无效、网络不可达、超时、TLS/认证失败、设备拒绝、协议不兼容、会话被远端关闭、未知错误。

设备档案：

- 成功连接后自动保存或更新显示名、主机/IP、最近成功调试端口、首次连接时间、最近连接时间、本机标记、主机身份引用。
- 支持编辑显示名、删除、手动重连；按最近连接时间倒序。
- 删除档案必须删除关联密钥/身份引用，且二次确认。
- 配对码只存在内存；成功、失败、取消、离开页面后立即清除。
- 私钥和配对身份必须由 Android Keystore 保护。
- 不保存 Shell 历史、终端输出、Logcat 或进程快照。

### 3. 设备概览

实现可取消、可刷新、可降级的设备概览：

- 品牌、制造商、型号、设备代号。
- Android 版本、SDK、构建信息、安全补丁日期。
- CPU ABI、可用核心数。
- 内存总量/可用量、内部存储总量/可用量。
- 电量、充电状态、温度（支持时）、运行时间、网络地址（支持时）。
- 静态信息首次进入时加载，支持手动刷新。
- 电池与内存仅在本页面前台时每 5 秒刷新；离开页面、切换设备、断开后必须停止。
- ROM 不提供或无法可靠解析的字段显示“设备未提供”，不可显示虚假数据。
- 命令和 ROM 差异必须封装在 core:adb 的能力层，不得散落在 UI。

### 4. Shell 终端

实现原始 ADB Shell 终端：

- 支持多行命令输入、执行、运行中取消、命令顺序记录、复制。
- 尽可能区分 stdout 和 stderr；底层协议无法区分时，清晰标记为合并输出，不能伪称已区分。
- 不设白名单，不静默修改或拦截用户命令。
- 首次进入显示一次风险说明；明显高风险命令可提示风险，但用户可继续执行。
- 默认超时 30 秒；用户可在当前会话调整至最多 5 分钟。
- 会话内容只在内存中保存；切换活跃设备或退出应用后清除。
- 终端输出上限 1 MiB；达到上限时丢弃最早内容并显示提示。
- 执行、取消、超时、断开和失败必须在 UI 中清晰标记。

### 5. 进程监控

实现只读进程页面：

- 显示进程名、PID、UID、状态，以及设备能够可靠提供的内存数据。
- 支持搜索、手动刷新、复制 PID、复制进程名。
- 不提供结束进程、Kill、强制停止、应用卸载、冻结或任何应用管理能力。
- 对不同 ROM 的 `ps` / `top` 输出进行能力探测、兼容解析和明确降级。
- 字段无法获得时留空并说明，不崩溃、不伪造。

### 6. Logcat

实现前台实时 Logcat：

- 用户明确点击“开始”后才抓取；离开页面、断开连接或点击停止时立即停止。
- 默认缓冲区：main、system、crash。
- 默认最低等级：Info。
- 默认格式：线程时间。
- 支持等级 Verbose 到 Fatal、缓冲区 main/system/crash/radio/events、关键字过滤、暂停/继续、清屏、复制可见内容。
- 内存上限为 10,000 行或 4 MiB，先达到者生效；丢弃最早内容并提示。
- 使用系统 Storage Access Framework 手动导出当前可见内容为 UTF-8 `.txt`。
- 不申请所有文件访问，不自动保存、不后台继续、不上传。
- Logcat 过滤、采集和停止必须不会卡住 UI 或泄漏协程/Socket。
